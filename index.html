HTML

  • The head element contains information about the webpage
  • Head elements are not seen by the end user
  • the body element represents the visible content seen by the end user
  • child elements are elements that are nested inside another element
  • an element is the opening and closing tags together
  • tags are the opening and closing lines of the code "<>"
  • HTML is the frame of a website

CSS

  • CSS is the visual representation of a website, like paint on a house
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • to view css changed on a website make sure to link the css and html files
  • the selector in css determines which sections will be changed in html

Git

  • Git is where repositories are stored
  • git is a useful version control program
  • Git is useful for editing code in pieces instead of the entire code all at once
  • Git is used to work on a project with others by making seperate branches
  • git status: checks what branch we are currently on
  • git checkout - switch to another branch and check it out into your working directory
  • git checkout -b branch-name: - creates a new branch and switches to it
  • git config --global user.name “[firstname lastname]” - set a name that is identifiable for credit when review version history
  • git config --global user.email “[valid-email]” -set an email address that will be associated with each history marker
  • git branch - list your branches. a * will appear next to the currently active branch
  • git branch [branch-name] - create a new branch at the current commit
  • git add [file] - add a file as it looks now to your next commit (stage)
  • git commit -m “[descriptive message]” - commit your staged content as a new commit snapshot

JavaScript

  • Javascript is the functionality of a website and how the user can interact with the website
